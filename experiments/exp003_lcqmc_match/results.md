## Experiment 003 — LCQMC 文本匹配

### 基本信息

- **日期**: 2026-06-22
- **模型**: bert-base-chinese
- **任务**: 判断两个句子的语义是否匹配（二分类）
- **数据**: 自生成 LCQMC 风格数据
- **数据量**: 157 训练对 / 34 验证对 / 34 测试对
- **设备**: RTX 3050
- **训练时长**: ~30 秒（5 epochs，数据极少）

### 结果

| 指标 | 值 | 说明 |
|------|-----|------|
| **Test Accuracy** | **0.4412** | 低于随机基线 |
| Precision | 0.2206 | |
| Recall | 0.5000 | |
| **F1 Score** | **0.3061** | |
| Avg Confidence | 0.7133 | |

### 分 Breakdown

| 类别 | 正确数 | 准确率 |
|------|--------|--------|
| 不匹配 (label=0) | 15/15 | 100% |
| 匹配 (label=1) | **0/19** | **0%** |

### 问题分析

**模型根本就没学会"匹配"——它只是学会了预测"不匹配"。**

| 原因 | 说明 |
|------|------|
| 数据量严重不足 | 157 对训练数据，对于 BERT 理解语义匹配远不够 |
| 类别不均衡 | 62% 正例，38% 负例，但没有大到让模型走极端 |
| 任务难度高 | 匹配需要理解"同义不同词"，比分类难得多 |
| 关键词投机失效 | 分类中靠关键词就能猜对，匹配没有这种捷径 |

### 全预测为"不匹配"的根源

157 对数据中，相似对的表达差异很大：
```
A: 人均消费多少      B: 一个人大概多少钱
A: 怎么和老板谈加薪  B: 如何向老板提加薪
```

模型没有足够的样本学到"人均消费 ≈ 一个人多少钱"这种语义等价关系。

### 对比：分类 vs 匹配

| 维度 | 新闻分类 | 文本匹配 |
|------|---------|---------|
| 难度 | 低 | **高** |
| 最小数据量 | ~100 条可达 85%+ | **~2000 对可能才够** |
| 学习方式 | 关键词匹配即可 | 需要语义理解 |
| 评估指标 | Accuracy 足够 | **需要 F1** |
| 训练时间 | 秒级 | 需要更多数据 |

### 发现

1. **匹配是比分类难一个量级的任务**——不能简单复用分类的思路
2. **数据量是决定性因素**——157 对对于匹配来说就像 20 条对于分类
3. **F1 指标在这里比 Accuracy 更有意义**——Accuracy 44% 隐藏了"全猜不匹配"的问题
4. **模板化匹配数据没有用**：训练集和测试集的负例分布不同，导致泛化失败
